注意

Android4.0.4で、ヘッダー固定にしている場合、左右のナビがMVの下にもぐってしまうことがありました。（岩倉自動車教習所）
slider proのせいか、ナビのところの背景にsvgを使用しているのも原因の可能性があります。
新しい機種では問題ありませんでした。

js,cssを読み込ませる

<script src="slider-pro/jquery.sliderPro.min.js"></script>
<link rel="stylesheet" type="text/css" href="slider-pro/slider-pro.css" media="all">

html

PC用画像をsrc属性に、スマホ用画像をdata-smallとdata-medium属性に入れてください。

<div class="keyVisual">
	<div id="kvSlider" class="slider-pro">
		<div class="sp-slides">
			<div class="sp-slide">
				<a href=""><img class="sp-image" src="dummy_img/mv01.jpg" data-small="dummy_img/mv01_sp.jpg" data-medium="dummy_img/mv01_sp.jpg" alt=""></a>
			</div>
			<div class="sp-slide">
				<a href=""><img class="sp-image" src="dummy_img/mv02.jpg" data-small="dummy_img/mv02_sp.jpg" data-medium="dummy_img/mv02_sp.jpg" alt=""></a>
			</div>
			<div class="sp-slide">
				<a href=""><img class="sp-image" src="dummy_img/mv03.jpg" data-small="dummy_img/mv03_sp.jpg" data-medium="dummy_img/mv03_sp.jpg" alt=""></a>
			</div>
		</div>
	</div>
</div>

js

$(function(){
	$('#kvSlider').sliderPro({
		width: '100%',
		aspectRatio:2.4,
		arrows: true,
		buttons: true,
		waitForLayers: true,
		fade: false,
		autoplay: true,
		autoplayDelay: 5000,
		autoplayOnHover: 'none',
		autoScaleLayers: false,
		keyboard: false,
		loop: true,
		slideDistance: 0,
		slideAnimationDuration: 700,
		visibleSize: '100%',
		fadeArrows: false,
		breakpoints: {
			767: {
				aspectRatio:1.809672386895476,
				forceSize: 'fullWidth'
			}
		}
	});
});
